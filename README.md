## 🔧 Tối ưu hóa

### JavaScript Optimization

<details>
<summary><b>🚀 Kỹ thuật được áp dụng</b></summary>

#### 1. IIFE Pattern (Immediately Invoked Function Expression)
```javascript
(function() {
  'use strict';
  // Code here is isolated
})();
```
- ✅ Isolated scope
- ✅ Faster garbage collection
- ✅ No global pollution

#### 2. Pre-defined Constants
```javascript
const EXPIRES_DATE = "2099-07-18T01:04:17Z";
const PRODUCT_ID = "locket.premium.yearly";
```
- ✅ Created once
- ✅ Memory efficient
- ✅ Faster access

#### 3. Native Loops vs Array Methods
```javascript
// ❌ Slow: .find() iterates entire array
const match = Object.keys(mapping).find(e => ua.includes(e));

// ✅ Fast: for...in with early break
for (const key in mapping) {
  if (ua.indexOf(key) !== -1) {
    // Do work
    break; // Stop immediately
  }
}
```
- ✅ 30-40% faster
- ✅ Early termination

#### 4. Pre-built Objects
```javascript
// ❌ Slow: Create every time
function process() {
  const data = { key: "value", ... };
  return data;
}

// ✅ Fast: Create once, reuse
const data = { key: "value", ... };
function process() {
  return data;
}
```
- ✅ No re-creation overhead
- ✅ Memory efficient

#### 5. Minimal I/O Operations
```javascript
// ❌ Slow: console.log is I/O operation
console.log("Debug:", data);

// ✅ Fast: No logging in production
// Removed all console.log calls
```
- ✅ Zero I/O overhead
- ✅ Faster execution

</details>

## ❓ FAQ

<details>
<summary><b>❓ Module có hoạt động trên mạng di động không?</b></summary>

✅ **Có**, module hoạt động trên cả WiFi và mạng di động (4G/5G).

Lưu ý:
- Đảm bảo VPN/Proxy đã kết nối
- Kiểm tra MITM đã bật
- Một số mạng di động có thể chặn HTTPS decryption

</details>

<details>
<summary><b>❓ Tại sao YouTube vẫn có quảng cáo?</b></summary>

Kiểm tra các điểm sau:

1. ✅ **MITM đã bật?** - Settings → MITM → ON
2. ✅ **CA Certificate đã install?** - Settings → General → About
3. ✅ **Hostname đã thêm?** - Kiểm tra `youtubei.googleapis.com`
4. ✅ **Module đã bật?** - Configuration → Module → Toggle ON
5. ✅ **Đã reload config?** - Restart app sau khi cài

Nếu vẫn không được:
- Xóa cache YouTube: Settings → Apps → YouTube → Clear Cache
- Khởi động lại iPhone/iPad
- Reinstall module

</details>

<details>
<summary><b>❓ Spotify bị lỗi "Premium not available"?</b></summary>

**Giải pháp:**

1. **Logout → Login lại** Spotify
2. **Clear cache**: Settings → Storage → Clear Cache
3. **Kiểm tra hostname**: `spclient.wg.spotify.com`
4. **Wait 5-10 phút** sau khi install module
5. **Restart Spotify app**

**Lưu ý:**
- Một số tính năng cần thời gian sync
- Offline download UI sẽ hiện nhưng không download được

</details>

<details>
<summary><b>❓ Locket không unlock được?</b></summary>

**Debug steps:**

1. **Kiểm tra hostname**: `api.revenuecat.com`
2. **Test**: Mở Locket → Settings → Restore Purchase
3. **Check logs**: 
   - Surge: Home → Log → Filter "locket"
   - Shadowrocket: Log tab
4. **Verify scripts loaded**: Configuration → Check script paths

Nếu script error:
- Update module bằng cách import lại URL
- Kiểm tra internet connection
- Script phải load được từ GitHub

</details>

<details>
<summary><b>❓ Module có thu thập dữ liệu cá nhân không?</b></summary>

Module này:

## 🐛 Troubleshooting

### YouTube Issues

<details>
<summary><b>🔴 Video không play được</b></summary>

**Nguyên nhân:**
- Timeout trên mạng chậm
- Pattern regex không match

**Giải pháp:**
1. Tăng timeout lên 30s (edit module)
2. Test trên WiFi trước
3. Check pattern matching trong logs
4. Thử disable module → reload → enable lại

</details>

<details>
<summary><b>🔴 PiP không hoạt động</b></summary>

**Giải pháp:**
1. **iOS Settings**: 
   - Settings → General → Picture in Picture → Allow
2. **YouTube Settings**:
   - YouTube → Settings → General → Picture in Picture → Enable
3. **Module check**:
   - Verify script đang chạy (check logs)
4. **Reinstall**:
   - Remove module → Reinstall

</details>

### Spotify Issues

<details>
<summary><b>🟡 Shuffle vẫn bị bắt buộc</b></summary>

**Lưu ý:** 
- Free account có giới hạn từ Spotify server
- Module unlock UI nhưng không bypass server-side limits

**Workaround:**
- Logout → Clear cache → Login lại
- Wait 10 minutes
- Restart app

</details>

### Locket Issues

<details>
<summary><b>🟢 "Restore Purchase" không hoạt động</b></summary>

**Debug:**

1. **Check pattern match**:
```
Pattern: ^https:\/\/api\.revenuecat\.com\/.*\/(receipts$|subscribers\/[^/]+$)
Test: https://api.revenuecat.com/v1/subscribers/xxxxx
```

2. **Check script loaded**:
- Logs should show: "Locket script executed"

3. **Verify MITM**:
```bash
# Test command (terminal)
curl -x http://localhost:port https://api.revenuecat.com/v1/subscribers/test
# Should return modified response
```

4. **Force refresh**:
- Kill Locket app completely
- Clear app cache (iOS Settings)
- Reopen Locket

</details>

### Common Issues

<details>
<summary><b>⚠️ Script không load được</b></summary>

**Nguyên nhân:**
- GitHub down hoặc bị chặn
- Network issue
- Script path sai

**Giải pháp:**

1. **Test GitHub access**:
```bash
ping raw.githubusercontent.com
```
